# reubeningber.com — AI Agent Guide

Personal blog and portfolio site for Reuben Ingber. Built with Astro 5, Tailwind CSS, and deployed to Cloudflare Pages.

## Tech Stack

- **Framework**: Astro 5 (static site generation, no SSR)
- **Styling**: Tailwind CSS v3
- **Content**: Astro Content Collections (Markdown files)
- **Images**: Cloudinary (cloud name stored in `PUBLIC_CLOUDINARY_CLOUD_NAME` env var)
- **Analytics**: Plausible (`PUBLIC_PLAUSIBLE_DOMAIN` env var) + Google Analytics (GA4 tag `G-8DBW26ND7K`, hardcoded in `BaseLayout.astro`)
- **Site URL**: `https://reubeningber.com`

## Key Commands

```bash
npm run dev      # local dev server
npm run build    # production build to /dist
npm run preview  # preview the build
```

## Content Collections

### Posts (`src/content/posts/`)

Blog articles. Frontmatter fields (`src/content/config.ts`):

| Field | Type | Required | Notes |
|---|---|---|---|
| `title` | string | yes | |
| `subTitle` | string | no | Shown under the title and used as excerpt in listings |
| `pubDate` | date | yes | Controls publish ordering and scheduling |
| `updatedDate` | date | no | |
| `draft` | boolean | no | Defaults `false`; drafts hidden in production |
| `image` | string | no | See Image Paths section below |
| `imageCredit` | string | no | Shown on hover over post image |
| `imageCreditUrl` | string | no | Makes imageCredit a link |
| `category` | string | no | Drives category pages; use consistent casing (e.g. "Fatherhood") |
| `description` | string | no | Used for meta description and RSS; falls back to first paragraph |
| `tags` | string[] | no | Used for meta keywords and schema |

### Naming convention

Post files are named `YYYY-MM-DD-slug-title.md`. The `pubDate` in frontmatter is the source of truth for ordering; the filename date is just for organization.

### Scheduled posts

Posts with a `pubDate` in the future are hidden automatically — the build filters them out. Set a future date to schedule a post.

### Field Notes (`src/content/field-notes/`)

Short-form entries: links, images, or embeds with optional commentary. Lives at `/field-notes/`. Paginated at 10 per page. Frontmatter fields:

| Field | Type | Required | Notes |
|---|---|---|---|
| `title` | string | yes | For `link` type, renders as an external link (↗) to `url` |
| `pubDate` | date | yes | Controls ordering and scheduling |
| `type` | `link` \| `image` \| `embed` | yes | Determines how the entry is rendered |
| `url` | string (URL) | for link | Target URL; title becomes a link opening in new tab |
| `image` | string | for image | Cloudinary path (e.g. `web_assets/photo.jpg`) or full URL |
| `embed` | string | for embed | Raw HTML embed code (iframe, etc.) |
| `draft` | boolean | no | Defaults `false`; drafts hidden in production |

The Markdown body is rendered as commentary below the media. Files follow the same `YYYY-MM-DD-slug.md` naming convention.

**Creating a log entry:**
```markdown
---
title: Some Article Worth Reading
pubDate: 2026-07-01
type: link
url: https://example.com/article
---

A sentence or two about why this is worth your time.
```

## Image Paths

All images go through Cloudinary. The `getImageConfig()` logic in `PostLayout.astro` handles three formats for the `image` frontmatter field:

1. **Full Cloudinary URL** (`https://res.cloudinary.com/...`) — path is extracted and re-optimized
2. **`web_assets/` path** (e.g. `web_assets/my-photo.jpg`) — preferred new format; resolves to `https://res.cloudinary.com/{cloudName}/image/upload/.../v1761245976/{path}`
3. **`assets/images/` path** — legacy local path, served as-is from `/public`

All Cloudinary images use a blur-up loading pattern (tiny blurred placeholder swapped for the full image on load).

The Cloudinary version token used throughout is `v1761245976`.

## Adding a New Post

1. Create `src/content/posts/YYYY-MM-DD-your-title.md`
2. Add frontmatter with at minimum `title` and `pubDate`
3. Write content in Markdown below the `---`

Example:
```markdown
---
title: My New Post
subTitle: An optional subtitle shown in listings
pubDate: 2026-06-16
category: Fatherhood
image: web_assets/my-photo.jpg
description: A short description for SEO.
tags: [parenting, kids]
---

Post content here.
```

## Layout & Styling Notes

- All pages inherit `BaseLayout.astro` — do not add analytics, OG tags, or structured data elsewhere
- The accent color is orange (`text-accent` = `text-orange-600`); primary text is dark (`text-primary`)
- All heading text is rendered `toUpperCase()` via JS in templates — write titles in normal case in Markdown
- The Rock Salt font (Google Fonts) is used only for the site logo/name in `Header.astro`
- Tailwind prose classes style Markdown body content in `PostLayout.astro`

## Analytics

Two analytics tools run in parallel via `BaseLayout.astro`:
- **Plausible**: privacy-first, rendered conditionally via `Plausible.astro` if `PUBLIC_PLAUSIBLE_DOMAIN` is set
- **Google Analytics (GA4)**: hardcoded tag `G-8DBW26ND7K`, always active in production

## Environment Variables

| Variable | Purpose |
|---|---|
| `PUBLIC_CLOUDINARY_CLOUD_NAME` | Cloudinary cloud name for image URLs |
| `PUBLIC_PLAUSIBLE_DOMAIN` | Domain string for Plausible analytics (optional) |
| `SHOW_DRAFTS` | Set to `"true"` in dev to preview draft posts |

## Navigation

- ARTICLES → `/articles/`
- FIELD NOTES → `/field-notes/`
- PHOTOS → `https://photos.reubeningber.com` (external)
- CONTACT → `/contact/`

## SEO & Structured Data

- `BaseLayout.astro` includes a `Person` schema for Reuben site-wide
- `index.astro` adds `WebSite` and `WebPage` schemas
- `PostLayout.astro` adds a `BlogPosting` schema per article
- Canonical URLs are computed from `Astro.site` + `Astro.url.pathname`
- OG and Twitter card meta tags are set in `BaseLayout.astro` via props
